Module Rdf_ds

module Rdf_ds: sig .. end
Datasets.
A dataset is composed of:
  • a default graph,
  • a set of named graphs,
  • a function to get a graph by its IRI.

exception Could_not_retrieve_graph of Iri.t * string
This exception indicates that a graph could not be retrieved. It can be raised by functions retrieving graphs, in a Rdf_ds.dataset structure. The URI of the graph and an error message must be provided.
val could_not_retrieve_graph : Iri.t -> string -> 'a
This function raises the Rdf_ds.Could_not_retrieve_graph exception.
type dataset = {
   default : Rdf_graph.graph; (*
The default graph.
*)
   named : Iri.Set.t; (*
The set of named graphs.
*)
   get_named : Iri.t -> Rdf_graph.graph; (*
The function to get a graph by its name (URI). The function must raise Rdf_ds.Could_not_retrieve_graph in case of error.
*)
}
A dataset.
val simple_dataset : ?named:(Iri.t * Rdf_graph.graph) list -> Rdf_graph.graph -> dataset
simple_dataset graph returns a dataset with graph as default graph.
named : can be used to specify named graphs. The get_named function is created from this closed list of named graphs and raise Rdf_ds.Could_not_retrieve_graph in case a required graph is not part of the list.
val dataset : ?get_named:(Iri.t -> Rdf_graph.graph) ->
       ?named:Iri.Set.t -> Rdf_graph.graph -> dataset
dataset graph returns a dataset with graph as default graph.
get_named : is the function to retrieve graph by name (URI). If it is not provided, the default function always raises Rdf_ds.Could_not_retrieve_graph.
named : is used to specify the sef of named graphs, but it does not create a get_named function.
