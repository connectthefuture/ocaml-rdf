Module type Rdf_sparql_http.S

module type S = sig .. end
The signature of the resulting module when buildting the binding.
type result 
val get : ?graph:Rdf_graph.graph ->
       base:Rdf_iri.iri ->
       ?accept:string ->
       Rdf_uri.uri -> Rdf_sparql_protocol.in_message -> result
get uri msg sends the Sparql query corresponding to the given Rdf_sparql_protocol.in_message with the GET method. uri is the complete uri of the service to connect to, i.e. not only the protocol, host and port, but also the path of the service, for example http://dbpedia.org/sparql.
graph : and base parameters are the same as in Rdf_sparql.execute.
accept : can be used to specify the Accept: header. Beware that the implementation handles only some content-types. This parameter can be used to require a response in, for example, the "application/rdf+xml" format.
val post : ?graph:Rdf_graph.graph ->
       base:Rdf_iri.iri ->
       ?accept:string ->
       Rdf_uri.uri ->
       ?query_var:string ->
       Rdf_sparql_protocol.in_message -> result
Same as Rdf_sparql_http.S.get but using the POST method. To be used when sending large queries.
query_var : allows to change the message query name send ("query" by default)
