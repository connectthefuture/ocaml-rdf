RDF terms and IRIs

RDF graph terms are represented by the Rdf_term.term type. These are RDF terms: IRI, literal or blank node. In the graph, the predicate of a triple can only be a IRI.

All strings are considered to be UTF8-encoded. The Rdf_utf8 module provides some functions to manipulate UTF8 strings.

The Rdf_term module defines functions to create and print RDF terms. For example, Rdf_term.mk_literal_int creates a literal RDF term from an integer, setting the IRI of the datatype to <xsd:integer>: 

# let int_lit = Rdf_term.mk_literal_int 42;;
val int_lit : Rdf_term.literal =
  {Rdf_term.lit_value = "42"; lit_language = None; lit_type = Some <abstr>}

# let term42 = Rdf_term.Literal int_lit;;
val term42 : Rdf_term.term =
  Rdf_term.Literal
   {Rdf_term.lit_value = "42"; lit_language = None; lit_type = Some <abstr>}

Let's get a string representation of the term: 

# print_endline (Rdf_term.string_of_term term42) ;;
"42"^^<http://www.w3.org/2001/XMLSchema#integer>

- : unit = ()

The Rdf_iri module defines the type Rdf_iri.iri to represent IRIs and contains functions to create IRIs and perform other manipulations: 

# let iri_s = "http://foo.bar.net/";;
val iri_s : string = "http://foo.bar.net/"

# let iri = Rdf_iri.iri iri_s;;
val iri : Rdf_iri.iri = <abstr>

# let iri2 = Rdf_iri.set_fragment iri (Some "id42");;
val iri2 : Rdf_iri.iri = <abstr>

# Rdf_iri.compare iri iri2;;
- : int = -1

# Rdf_iri.string iri2;;
- : string = "http://foo.bar.net/#id42"

The Rdf_rdf module defines some commonly used IRIs: datatypes, classes, property names, ...: 

# Rdf_iri.string (Rdf_rdf.xsd_integer);;
- : string = "http://www.w3.org/2001/XMLSchema#integer"

# Rdf_iri.string (Rdf_rdf.xsd_double);;
- : string = "http://www.w3.org/2001/XMLSchema#double"

# Rdf_iri.string (Rdf_rdf.rdf_about);;
- : string = "http://www.w3.org/1999/02/22-rdf-syntax-ns#about"
