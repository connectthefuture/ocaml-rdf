Download

The code is available on Github.

Use git to get the current version:

$ git clone https://github.com/zoggy/ocaml-rdf.git

Then read the Installation section.

Releases
  • 0.9.0 [2014-09-29]:
    • use -safe-string compilation option (and require OCaml >= 4.02),
    • provide rdf.ppx package for [%sparql ] and {sparql|...|sparql} syntax,
    • new Rdf_lwt library implementing Sparql protocol if Cohttp with lwt is installed,
    • Support for Sparql protocol initiated by Aranud Parant then made more generic to support various HTTP implementations and the whole protocol,
    • Support for 4store access through Sparql protocol (by Arnaud Parant),
    • Fix parsing of IRIs.
  • 0.8.0 [2014-01-21]:
    • Change Rdf_rdf.dc to http://purl.org/dc/elements/1.1/,
    • Add convenient functions to create RDF vocabularies in Rdf_rdfs,
    • Fix: handle more than one FROM <iri> directives in sparql query according to specification (merge referenced graphs in a new graph in memory).
  • 0.7.1 [2013-09-27]:
    • fix MySQL backend: handle empty engine option and set DELAY_KEY_WRITE=1 only for MyISAM engine.
  • 0.7.0 [2013-09-24]:
    • Rdf_iri now replaces Rdf_uri,
    • Rdf_sparql.parse_from_* renamed to query_from_*,
    • Rdf_uri.Invalid_url renamed to Invalid_uri,
    • Rdf_ttl.from_* do not return a graph, as the given one is modified in place,
    • The base argument of Rdf_ttl.from_* and Rdf_xml.from_* is now optional; if not given, the graph name is used as base,
    • Add namespace modules: Rdf_foaf , Rdf_earl , Rdf_foaf , Rdf_foaf , Rdf_rdfs ,
    • Partial fix: implement simple version of Sparql's builtin funs LCASE and UCASE,
    • Upgrade turtle parser/printer to W3C RC of 2013/02/19, passes all tests of the Turtle Test Suite except the test containing an invalid UTF-8 character,
    • Do not store namespaces in the graph itself (that was a very bad idea); add functions to add and remove namespaces for a graph,
    • Functor to create new backends now allows to keep a backend-specific representation of terms during Basic Graph Pattern evaluation; this is used to optimize database backends,
    • New "mysql2" backend, faster for queries but slower for insertions,
    • MySQL backend now supports the "engine" option, default is INNODB, but it can be "MyISAM" (faster, but does not support transactions),
    • Remove graph debug dump which was performed at each sparql query evaluation,
    • Rdf_node renamed to Rdf_term ,
    • Rdf_node.node renamed to Rdf_term.term ,
    • Rdf_node.NSet renamed to Rdf_term.TSet ,
    • In triples, predicates are now IRIs and not any term.
    • + lots of bug fixes.
  • 0.6.0 [2013-08-13]:
    • Add support of Sparql 1.1 queries,
    • Add missing dependencies in META file,
    • Various fixes.
    • See the complete list.
  • 0.5 [2013-05-22]:
    • Add Turtle parser,
    • Rdf_iri.iri now raises Rdf_uri.Invalid_url instead of Neturl.Malformed_URL,
    • See the complete list.
  • 0.4 [2013-03-19]: Use string concatenation instead of sprintf; change default graph kind: memory -> mem; linkall option.
  • 0.3 [2013-01-21]: Some fixes. Add graph merging.
  • 0.2 [2012-10-09]: Improve performance by using prepared queries. New backend: Postgresql.
  • 0.1 [2012-09-26]: Initial release.
